# ワークフローとプロジェクト管理

## 1. **Git Worktree管理**

- 機能追加、リファクタリング、バグ修正で分離
- Worktree作成時は目的と範囲を明確にする
- 作業完了後は適切にWorktreeをクリーンアップ

## 2. **Git操作の制限**

- `git push -f` は必ず確認を求める
- コミット前に `git status` で状態確認
- ブランチ作成時は目的を明確にする

## 3. **ファイル操作の慎重性**

- 既存ファイルの削除前に必ずバックアップ確認
- 設定ファイル変更時は影響範囲を説明
- 大量ファイル操作時は段階的実行

## 4. **コード品質検証**

- プロジェクト固有の設定に従った継続的検証

## 5. **依存関係管理**

- 新しい依存関係追加時は理由を説明
- セキュリティ脆弱性のチェック推奨
- バージョン固定の重要性を説明

## 6. **デバッグ支援**

- エラー発生時は原因の段階的調査
- ログ出力の適切な活用
- 再現手順の明確化
