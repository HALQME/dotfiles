# プロジェクト固有のルールとコンテキスト

## 1. **アーキテクチャ原則**

- マイクロサービス vs モノリス判断基準
- API設計のRESTful原則遵守
- データベース設計の正規化レベル

## 2. **技術スタック制約**

- 使用している言語、ランタイム、パッケージマネージャーの統一
- 承認済み技術スタックのみ使用
- 新技術導入時は事前相談必須
- レガシーシステムとの互換性考慮

## 3. **環境別設定**

- 開発・ステージング・本番環境の分離
- 環境変数による設定管理
- シークレット管理のベストプラクティス

## 4. **チーム規約**

- コードレビュー必須項目
- ドキュメント更新タイミング
- 緊急時対応プロセス
